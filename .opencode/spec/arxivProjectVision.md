
# arXiv 论文项目 · 项目愿景 v1.0

## 要做什么
- 每天抓取 arXiv 最新人工智能领域论文 20 篇
- 只抓取 AI 相关论文（优先 `cs.AI`，可包含 `cs.LG`、`cs.CL`、`cs.CV`、`cs.RO`、`cs.MA` 等相关分类）
- 用 Agent 分析内容，分析维度包括：论文摘要、研究方向、核心方法、主要贡献、应用场景
- 输出结构化结果，字段包括：论文标题、作者、arXiv 链接、PDF 链接、英文摘要、中文摘要、研究方向、关键词
- 将每日结果保存到 `paperReview/` 目录，便于后续查阅和复用

## 不做什么
- 不做用户系统和权限管理
- 不做 Web 界面（初期仅命令行/脚本运行）
- 不做复杂的历史数据检索与推荐系统（初期仅保存每日结果）
- 不做实时推送和通知功能
- 不做论文全文精读、复现实验与代码级验证

## 边界 & 验收
### 边界
- 仅处理 arXiv 上公开可访问的论文
- 每日运行一次，不做实时监控
- 仅分析论文标题、摘要、分类和基础元数据，不下载全文做深度解析
- 使用公开网页内容进行采集，不依赖私有数据库或封闭接口

### 验收标准
- 能成功从 arXiv 抓取 20 篇最新 AI 相关论文
- 能为每篇论文生成符合要求的结构化结果
- 每个条目的分析内容完整、准确、有价值
- 系统能稳定运行，无崩溃或严重错误

## 怎么验证
- 每日运行后，手动抽查 3-5 篇论文的分析结果
- 验证条目字段是否完整，链接是否可访问
- 验证中文摘要和研究方向判断是否准确、有意义
- 验证是否成功抓取了 20 篇 AI 相关论文

## 与现有 Agents / Skills 的对应关系
- 列表采集阶段：由 `arxiv_list_collector` Agent 负责，目标是从 arXiv 最新列表中提取论文基础信息；对应 Skill 为 `arxiv-list-collector`
- 详情补全阶段：由 `arxiv_detail_collector` Agent 负责，目标是补全英文摘要、评论和基础元数据；对应 Skill 为 `arxiv-detail-collector`
- 领域筛选阶段：由 `arxiv_domain_filter` Agent 负责，目标是确认论文是否属于人工智能领域并补充研究方向标签；对应 Skill 为 `arxiv-domain-filter`
- 中文摘要阶段：由 `arxiv_summary_writer` Agent 负责，目标是生成高质量中文摘要，便于快速阅读；对应 Skill 为 `arxiv-summary-writer`
- 结果整理阶段：由 `arxiv_manager` Agent 负责，目标是去重、排序、格式化并保存到 `paperReview/`；对应 Skill 为 `arxiv-manager`
- 统一编排入口：完整流程通过 `arxiv-papers` Skill 串联以上步骤，作为 arXiv 项目的统一入口

## 建议执行流程
1. 先执行列表采集，获得当日候选论文
2. 再补全论文详情，形成可分析的基础数据
3. 然后做 AI 领域过滤，剔除不相关论文
4. 接着生成中文摘要和研究方向信息
5. 最后统一整理、去重并落盘保存
