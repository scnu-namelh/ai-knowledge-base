# GitHub Trending Skill — 设计规约

## 调用方式

**Model-invoked**（保留 description）。此技能被 collector agent 和人类用户共同触发。collector agent 权限受限（无 Bash、无 Write），因此 skill 的所有步骤必须仅依赖 WebFetch 完成，输出通过 agent 响应消息返回，而非写入文件。

## Description 设计

覆盖 4 个语义分支，每个分支有多个自然语言变体但归为一个概念：

| # | 分支 | 用户可能说 |
|---|------|-----------|
| 1 | **采集**（collect） | 采集/抓取/爬取/拉取/获取/收集 GitHub 热门/趋势/热点/Trending |
| 2 | **查看**（browse） | 查看/看看/查一下/浏览 GitHub 今日热门/趋势/Trending/热点 |
| 3 | **更新**（refresh） | 更新/刷新/同步 GitHub 热门数据/采集数据/Trending 数据 |
| 4 | **推荐**（discover） | 推荐/发现 GitHub 热门项目/好项目/开源榜单/排行榜 |

Description 结构：**主引导词 + 动作分支 + 语义靶点 + 表达示例**

```
热门 — 采集/查看/更新/推荐 GitHub Trending 项目数据。触发场景：采集 GitHub 热门、查看 Trending、更新热门数据、推荐开源项目、抓取趋势、爬取热点、刷新采集、浏览榜单。
```

## 主引导词

- **热点** — 主线锚点，"当下大家关注的东西"
- **去重** — 步骤 4 的质量锚点

"采集" 不单独做引导词，因为它是动作而非概念——嵌入 description 的动作分支中。

## 信息层级

- `SKILL.md`：步骤（主）+ 输出格式（内联参考）
- 步骤少（7 步），无需折叠

## 步骤完成准则

| 步骤 | 完成准则 |
|------|----------|
| 1. 搜索热门仓库 | GitHub Trending 页面完全加载并解析 |
| 2. 提取信息 | 每个仓库提取了 name、url、stars、language、topics、description |
| 3. 过滤 | 所有非 AI/LLM/Agent 项目和 Awesome 列表已移除 |
| 4. 去重 | 每个候选与全部历史文件逐项比对完毕 |
| 5. 撰写摘要 | 每个项目有符合「项目名+做什么+为什么关注」格式的中文摘要 |
| 6. 排序取前 | Star 降序、精确截取 15 条 |
| 7. 输出 | 输出数据已格式化并通过 JSON schema 校验 |

## 修剪清单

- ☑ **无重复**：description 中 4 个分支是不同概念，非同一概念的不同说法
- ☑ **无非操作**：每行都改变默认行为
- ☑ **无否定**：正向指令替代禁止句式
- ☑ **无沉积**：全部内容活跃可用
- ☑ **无扩散**：62 行，长度合理

4. 增加步骤完成准则
